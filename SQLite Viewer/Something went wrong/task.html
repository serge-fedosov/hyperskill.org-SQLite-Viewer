Description

What if something goes sour? We need to account for unplanned scenarios to avoid application errors. Let's add error handlers to our application.

Objectives

Pop up a dialog window with an error message if something is wrong. You can use the following statement:

JOptionPane.showMessageDialog(new Frame(), "ERROR MESSAGE");

It will open up a new frame with an error message. A user will have to close it to continue using the application.

Here are the cases to account for:

  • If a user enters a wrong file name, pop up a dialog window with the File doesn't exist! error message.
  • If a user enters a wrong SQL query, pop up a dialog window with an SQL error message. We won't check the text in the tests.

Don't forget that a user shouldn't be able to choose tables from the Combobox, to input a query to the query field, and click the execute button if the program couldn't establish a connection with the database or the database file name is empty.

Use setEnabled() method to enable or disable the components.
